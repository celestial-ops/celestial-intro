Celestial

provisioning server

Created by Ronen Narkis / @narkisr

We use Puppet/Chef for configuration managment

We deploy our code using Capistrano/Fabric

We run against EC2/VMware/...

Now we need to

functional requirements

  • Have an API and UI
  • Allow users to self service
  • Create systems automatically (nightly, auto-scaling)
  • Support multiple platforms as transparently

Technical requirements

  • Integrate desperate tools/APIs into a cohesive system
  • Prevent collisions between tools
  • Audit systems changes
  • Easy triaging when things go wrong
  • Update external systems automatically (DNS, monitoring, routing? etc..).

Enter Celestial

Best practices baked in

  • Unified Access via documented Restful API
  • Complete Auditing of all actions (via Graylog2/Kiabana)
  • Asynchronous execution with distributed locking
  • Role and Quota management
